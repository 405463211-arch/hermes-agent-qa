## 1. 上游 v0.13.0 带来的关键新特性（必须保留）

### 1.1 `_POLISHED_TOOLS` ACP 渲染重构（PR `ef9a08a87`）

- **作用**：把 ~30 个工具列入"polished tools"白名单（包括 `terminal`、`patch`、`web_search` 等）。这些工具的 `raw_output` 改为 `None`，UI 用 content blocks 渲染。
- **位置**：`acp_adapter/tools.py:1162`
- **影响**：所有 ACP 客户端（Zed 等）走 polished 工具的 turn 不再回填 raw JSON，UX 更干净。
- **本地适配**：测试断言需要从 `raw_output != None` 改为 `raw_output is None + content[0].text`。已修 `tests/acp/test_mcp_e2e.py::test_prompt_with_tool_calls_emits_acp_events`（commit `97c11cf48`）。

### 1.2 DeepSeek V4 Pro thinking-mode 兼容（PR #17341）

- **作用**：DeepSeek V4 Pro API 拒绝空字符串 `reasoning_content`，必须用单空格 `" "` 占位。
- **位置**：`run_agent.py::_copy_reasoning_content_for_api`（~L9466 / L9507）
- **影响**：所有走 DeepSeek 系（v4-flash / v4-thinking / reasoner）的 subagent / 多轮 turn 不再被 HTTP 400 切断。
- **本地适配**：3 个 `test_*_backfills_empty_reasoning_*` 测试断言从 `""` 改为 `" "`。已修（commit `b0e9fa568`）。

### 1.3 `is_truthy_value()` 全局环境变量守卫（utils.py）

- **作用**：统一所有"truthy 字符串"判定逻辑，**修复 YOLO 安全漏洞**——`HERMES_YOLO_MODE=0/false/off` 之前会因 `os.getenv` 返回非空字符串而 truthy。
- **位置**：`utils.py:20` + `tools/approval.py:893/1023`（两处 `HERMES_YOLO_MODE` 调用）
- **影响**：**安全回归**——v0.12.0 era 写"HERMES_YOLO_MODE=0 关闭 YOLO"是无效的，命令依然 bypass。
- **本地适配**：合并时"取本地" `tools/approval.py` 把这个守卫静默丢了，被 Batch 2 测试发现并修回（commit `a6a3f6da7`）。

### 1.4 `clear_session()` 唤醒已阻塞 entries

- **作用**：会话边界清理时，立即把 `_gateway_queues` 里所有阻塞中的 `_ApprovalEntry` 设为 `result="deny"` 并 `event.set()`。
- **位置**：`tools/approval.py::clear_session` 末尾的 `for entry in entries:` 循环
- **影响**：`/stop` 或会话切换不再让 gateway approval waiter idle 到自身 timeout，旧 run 立刻 unwind。
- **本地适配**：合并时"取本地"再次静默丢了这 5 行，被 Batch 3 测试发现并修回（commit `bf2a8899b`）。

## 5. 给下一次合并者（你 / 未来 agent）的话

### 5.1 第一原则：`tools/approval.py` 是 Mode D 高发地带

本次和 v0.12.0 都在这个文件出现过静默合丢。下次合并这个文件时，**不要简单 `git checkout --ours`**，而是：

```bash
# 1. 列出本地（HEAD）和上游（v<tag>）的所有 def/class diff
diff <(git show HEAD:tools/approval.py | grep -nE "^(def |class )") \
     <(git show v2026.5.X:tools/approval.py | grep -nE "^(def |class )")

# 2. 对每个上游新增/修改的函数手动 review
# 3. 决定：取本地保 bug fix，但每一个上游新增/修改的 helper 都要手动 cherry-pick 进来
```

### 5.2 第二原则：worktree 上游对比是 **唯一** 可靠的真回归侦测

```bash
git worktree add /tmp/check-vX v<tag>
ln -sf /Users/yhn/PyCharmMiscProject/hermes-agent-main/venv /tmp/check-vX/venv
cd /tmp/check-vX && bash scripts/run_tests.sh <failed_tests> > /tmp/upstream.log 2>&1
comm -23 <(sort local_failed.txt) <(sort upstream_failed.txt)
```

本次省下了大量时间和误判。

### 5.3 第三原则：分批跑测试，写到文件，不要 pipe

```bash
bash scripts/run_tests.sh <subset> > /tmp/batch.log 2>&1   # OK
bash scripts/run_tests.sh <subset> 2>&1 | grep FAILED      # 在某些情况下会挂
```

实测全量跑 `bash scripts/run_tests.sh`（无目录限制）会在某个测试套挂住超 25 分钟。分批 + 写文件解析最稳。

### 5.4 第四原则：`audit_merge_loss.sh` 是兜底，不是终点

- 自动 baseline 选择经常拿错（找 `snapshot:` substring 命中无关 commit）
- 手工 `bash scripts/audit_merge_loss.sh 73bf3ab1b`（v0.12.0 上游 commit）效果更好
- 本次审计只发现版本字符串"良性 loss"，**两个真静默合丢都是测试发现的**，不是审计

→ 下次升级 audit_merge_loss.sh：增加"对所有 .py 文件做函数级 diff"模式。

### 5.5 第五原则：CI 类型 4 模式诊断仍然适用

| Mode | 本次 v0.13.0 例子 |
|---|---|
| A 上游自破 | 14 个：lmstudio / stepfun / codex / cron / matrix / discord / anthropic_adapter / bedrock / unsupported / reasoning_content_replay 等 |
| B Fork 缺 secret | 0 个本次（v0.12.0 修过的 auto-fix-main gate 还在生效） |
